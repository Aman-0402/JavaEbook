📗 6. Object-Oriented Programming (OOP)

🔷 Class and Object

A class is a blueprint for objects. An object is an instance of a class.

class Student {
    String name;
    int age;
}
public class Test {
    public static void main(String[] args) {
        Student s = new Student();
        s.name = "Aman";
        s.age = 20;
        System.out.println(s.name + " is " + s.age + " years old.");
    }
}

🔷 Constructor (Default & Parameterized)

Constructors initialize objects. Java provides a default constructor if none is defined.

class Car {
    String model;
    Car(String m) { // Parameterized
        model = m;
    }
    Car() { // Default
        model = "Unknown";
    }
}

🔷 this Keyword

this refers to the current object and is used to distinguish between instance variables and parameters.

class Book {
    String title;
    Book(String title) {
        this.title = title;
    }
}

🔷 Method Overloading

Multiple methods with the same name but different parameters.

class MathUtil {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
}

🔷 Static Members

static variables/methods belong to the class rather than objects. They can be accessed without creating objects.

class Counter {
    static int count = 0;
    Counter() {
        count++;
    }
}

🔷 Access Modifiers

ModifierScope
privateAccessible within the same class
defaultAccessible within the same package
protectedAccessible in the same package and subclasses
publicAccessible from anywhere

🔷 Encapsulation

Wrapping data and methods into a single unit and restricting direct access to some components.

class Employee {
    private String name;

    public void setName(String n) {
        name = n;
    }

    public String getName() {
        return name;
    }
}
